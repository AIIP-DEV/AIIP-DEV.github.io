스케줄러 생성

  • 워크플로우를 선택하여 스케줄러를 생성 합니다.
​

워크플로우 선택​

스케줄러를 등록할 워크플로우를 선택 합니다.

  1. 검색 - 워크플로우를 검색
  2. 스케줄러를 등록할 워크플로우 선택
  3. 페이지 선택
  4. 취소 - 워크플로우 선택을 취소하고 팝업을 닫음
  5. 등록 - 선택한 워크플로우로 스케줄러 등록 화면으로 이동
​

스케줄러 등록하기​

​

1. 헤더​

  1. Chevron - 이전 화면으로 이동
​

2. 기본 정보 설정​

  1. 스케줄러 이름 입력
  2. 실행 이력 보관주기 선택 - 1일, 1주일, 1개월, 3개월, 6개월, 1년 중 선택
​

3. 스케줄 설정​

3-1. 기본방식​

  1. 입력 방식 - 기본방식, Cron 표현식 중 선택
  2. 시작 - 스케줄러 시작 일시 입력 (스케줄러 자체의 시작시간으로, job의 시작시간이 아님)
  3. 반복 주기 - 분, 시간, 일, 요일, 개월 단위로 반복 주기 입력
  4. 종료
    • 날짜 지정 - 지정된 날짜에 종료
    • 횟수 반복 - 지정된 횟수만큼 반복 후 종료

3-2. Cron 표현식​

  1. 입력 방식 - 기본방식, Cron 표현식 중 선택
  2. Cron 표현식 입력 - 최소 분단위로 설정 가능
  3. 시작 - 스케줄 시작 날짜 선택
  4. 종료 - 스케줄 종료 날짜 선택
  5. 등록 표준시간대 - 사용자가 계정정보에 지정한 표준시간대

3-3. 스케줄 동작 방식​

  1. 시작 - 스케줄러 자체의 시작시간으로, job의 시작시간이 아님
  2. 기본방식은 cron식과 동일하게 동작
    • ex) 10분 간격 → 0 /10 * *
  3. 항상 0분을 기준으로 하여 스케줄링됨 (cron 특징)
    • ex) 12:03부터 시작하며, 6분단위로 cron식을 짤 경우 → 12:06부터 수행
    • 2에 의해 기본방식인 경우에도 동일하게 동작
​

4. 알림 설정​

  1. 알림 대상 이벤트 - 알림 이벤트를 발생시킬 대상을 지정
    • 스케줄 시작 시 - 스케줄 시작시 알람 발생
    • 스케줄 종료 시 - 스케줄 종료시 알람 발생
    • 노드 별 - 워크플로우 노드 별 시작/종료 알람 발생
    • Drift Index "경고" 발생 시 - Drift Index 값이 0.25 초과일 때 알람 발생
    • Drift Index "주의" 발생 시 - Drfit Index 값이 0.1 초과, 0.25 이하일 때 알람 발생
  2. 알림 채널 - 스케줄 알림 채널 설정/해제
정보

서비스 관리자가 Admin 콘솔에서 알림 관리에 등록한 내용이 알림 채널에 표시 됩니다.

​

5. 변수 추가​

  1. key값 추가 - 변수 추가
  2. Key - 변수의 키 입력
  3. Value - 변수의 값 입력
​

6. 취소/등록​

  1. 입력한 내용을 취소하고 이전 화면으로 이동
  2. 입력한 내용으로 스케줄러 등록

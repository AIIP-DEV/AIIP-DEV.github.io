상세 화면

  • 프로젝트에 포함된 워크플로우, 샌드박스, 모델, 공유 범위를 프로젝트(나만보기)로 저장한 애셋을 관리하고 스케줄러를 설정할 수 있습니다.
​

1. 헤더​

  1. Project - 프로젝트 목록으로 이동
  2. 스케줄러 - 스케줄러 목록으로 이동
  3. 저장된 애셋 - 저장된 애셋 목록으로 이동
  4. 구성원 - 구성원 권한 편집 팝업 표시
  5. History - 프로젝트 수정 내역 팝업 표시
  6. More Actions
    • 기본정보 편집 : 프로젝트 수정 화면으로 이동
    • 프로젝트 삭제 : 프로젝트 삭제 수행
정보

권한에 따라 헤더 매뉴가 비활성화 되거나 표시되지 않는 항목이 있습니다.
프로젝트 삭제는 프로젝트 구성요소(워크플로우, 샌드박스, 모델)를 먼저 삭제해야 삭제가 가능 합니다.

​

2. 탐색​

  1. 보기 모드 - 카드 보기 모드와 리스트 보기 모드에 대해서 토글 수행
  2. 검색창 - 입력된 검색어로 검색 수행
  3. 생성자 필터 - 워크플로우를 생성한 사용자로 검색 수행
  4. 소팅 필터 - 최신 업데이트 순, 제목 오름차순, 제목 내림차순으로 소팅 수행
  5. 복원 - 선택한 파일로 복원 수행
​

3. 워크플로우 목록​

  1. 프로젝트에 작성된 워크플로우 수 표시
  2. 워크플로우 목록을 열거나 닫음
  3. 워크플로우 생성 화면으로 이동
  4. 북마크 설정/해제
정보

북마크 설정 시 우선적으로 정렬되어 리스트 상위에 표시 됩니다.

  1. 워크플로우 선택
  2. 워크플로우의 More Actions 메뉴
    • 편집 - 워크플로우 상세 화면으로 이동
    • 복제 - 워크플로우 복제 수행
    • 삭제 - 워크플로우 삭제 수행
정보

편집 시 Editor 권한이 없을 경우 읽기 전용 모드로 진입 합니다.

  1. 워크플로우에 스케줄러가 포함되어 있으면 아이콘 표시
​

4. 샌드박스 목록​

  1. 프로젝트에 속한 샌드박스 수를 표시
  2. 샌드박스 목록을 열거나 닫음
  3. 샌드박스 View 새로고침
  4. 신규 샌드박스 생성
  5. 북마크 설정/해제
정보

북마크 설정 시 우선적으로 정렬되어 리스트 상위에 표시 됩니다.

  1. 샌드박스 의 상태 및 변경 확인

    • Pending : 샌드박스 환경 생성 대기 중
    • Waiting : 샌드박스 환경 생성 중
    • Running : 샌드박스 환경에 접속 가능한 정상 실행중인 상태
    • Stop : 샌드박스 환경이 중지 상태이며, 접속 불가능.

  2. 샌드박스의 More Actions 메뉴

  • 샌드박스가 Stop 상태일 경우
    • 시작 - 샌드박스를 실행시켜 샌드박스를 Running 상태로 변경
    • 삭제 - 샌드박스 삭제 수행
  • 샌드박스가 wait 상태일 경우
    • 삭제 - 샌드박스 삭제 수행
  • 샌드박스가 Running 상태일 경우
    • 중지 - 샌드박스 중지
      • 샌드박스 중지 시 해당 샌드박스 이미지 백업 후 시작 시 해당 백업 이미지 사용
    • 샌드박스 애셋 저장 - 실행중인 샌드박스를 에셋으로 저장할 수 있는 팝업 표시
      1. 애셋 이름 - 저장할 샌드박스 애셋의 이름 입력
      2. 소스 포함 여부 - 샌드박스에 포함된 소스를 함께 저장할지 설정
      3. 설명 - 저장할 샌드박스 애셋의 상세 설명 입력
      4. 공유범위 설정 - 저장할 샌드박스 애셋의 공유 범위 설정
      5. 취소 - 샌드박스 애셋 저장을 취소하고 대시보드 메인 화면으로 돌아가기
      6. 저장 - 입력한 내용으로 샌드박스 애셋 저장 진행
    • 삭제 - 샌드박스 삭제 수행
정보

샌드박스 중지 시 해당 샌드박스 이미지를 백업하고, 해당 샌드박스 다시 시작을 하면 이전 중지 때 백업 된 이미지를 불러옵니다.

  • 삭제 - 샌드박스 삭제 수행
​

5. 모델 목록​

  1. 프로젝트에 작성된 모델 수 표시
  2. 모델 목록을 열거나 닫음
  3. 북마크 설정/해제
정보

북마크 설정 시 우선적으로 정렬되어 리스트 상위에 표시 됩니다.

  1. 모델 이름으로 AI Runtime 배포 표시 (회색일 경우 미 배포, 파란색일 경우 배포 됨)
  2. 모델의 More Actions 메뉴
    • AI Runtime 이동 - 선택한 모델이 Serving 중일 경우에만 표시. AI Runtime 서비스의 모델 배포 현황 페이지로 이동
    • 편집 - 모델이 작성되어 있는 워크플로우 상세 화면으로 이동
정보

AI Runtime 이동 시 배포가 삭제되었을 경우 오류 메시지가 발생할 수 있습니다.
편집 시 Editor 권한이 없을 경우 읽기 전용 모드로 진입 합니다.

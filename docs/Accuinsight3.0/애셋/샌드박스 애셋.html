샌드박스 애셋

  • 전사 레벨로 공유된 샌드박스 애셋을 조회하고 관리합니다.

샌드박스 애셋 화면​

1. 보기 모드​

카드 뷰와 리스트 뷰를 선택할 수 있습니다.

  • 샌드박스 애셋 카드

  • 샌드박스 애셋 리스트

​

2. 검색​

검색어를 입력 하면 애셋 이름, 프로젝트 이름, 공유자 이름을 대상으로 검색된 결과를 표시합니다.

​

3. 생성자 필터​

모델을 생성한 사용자로 검색할 수 있습니다.

​

4. 공개 필터​

공개 설정에 따른 애셋을 표시합니다.

  • 전체공개
  • 프로젝트 내 공개
  • 나만보기
​

5. 애셋 구성요소 필터​

선택한 구성요소를 포함한 애셋만 표시합니다.

  • "Jupyter LAb", "R Studio" 선택 가능 합니다.
​

6. 소스 필터​

소스 포함의 여부에 따른 애셋을 표시합니다.

​

7. 정렬​

  • 최신 업데이트 순 : 가장 최근에 업데이트된 애셋 순으로 정렬합니다.
  • 제목 오름차순 : 애셋 이름의 오름차순 순으로 정렬합니다.
  • 제목 내림차순 : 애셋 이름의 내림차순 순으로 정렬합니다.
​

8. More Actions​

카드 뷰 More Actions​

카드에 마우스 커서를 올리면 more actions 버튼이 표시되며, 샌드박스 애셋은 다음과 같은 메뉴를 제공합니다.

  • 샌드박스 시작 : 지정된 샌드박스를 시작합니다.

    • 샌드박스 시작 버튼 클릭 시 Sandbox 분석 환경 설정을 할 수 있는 페이지로 이동합니다.
      1. 분석환경 이름 : 생성할 Sandbox 분석 환경의 이름을 입력합니다. 필수로 입력해야 합니다.
      2. 분석 환경 : 분석 환경으로 사용할 샌드박스를 표시합니다. 샌드박스 애셋에서 실행하기로 한 샌드박스가 기본값으로 표시됩니다.
      3. 프로젝트 선택 : Sandbox를 실행할 프로젝트를 선택합니다. 필수로 선택해야 하며 선택 시 자원 설정과 Sandbox를 실행할 워크플로우를 선택할 수 있습니다.
      4. 프로젝트를 선택을 하면 Sandbox의 자원을 설정할 수 있습니다.
      5. 프로젝트를 선택하면 워크플로우를 선택할 수 있습니다.
      6. 취소 : Sandbox 분석 환경 설정을 취소하고 저장된 애셋 화면으로 이동합니다.
      7. 분석 환경 생성 : 입력한 값으로 Sandbox를 생성합니다.

  • 삭제 : 샌드박스 애셋을 삭제 합니다. 삭제 권한이 있을 때에만 표시 됩니다.

  • 수정 : 샌드박스 애셋을 수정 합니다. 수정 권한이 있을 때에만 표시 됩니다.

    • 수정 클릭 시 아래와 같은 팝업 창에서 수정 내용 입력

  1. 애셋 이름 수정
  2. 설명 : 샌드박스 애셋 설명 수정
  3. 공유범위 설정
  • 나만 보기 : 소스 저장소를 생성한 사용자만 공개
  • 프로젝트 내 공개 : 소스 저장소를 생성한 사용자가 포함된 프로젝트의 구성원에게 공개
  • 전체 공개 : 전체 구성원에게 모두 공개
  1. 산업군 수정
​

리스트 뷰 More Actions​

리스트의 more actions 버튼은 항상 표시되며, 카드 뷰와 동일한 메뉴를 제공 합니다.

​

다중 선택 More Actions​

여러개의 항목을 선택해서 More Actions를 실행할 수 있는 기능을 제공 합니다.

  • 카드 뷰 다중선택

    카드에 마우스 커서를 올리면 선택 체크박스가 표시됩니다. 한개, 여러개를 선택 했을 때 삭제 버튼이 표시 됩니다.

  • 리스트 뷰 다중선택

    리스트의 선택 체크박스는 항상 표시되며, 선택한 항목의 개수에 따라 표시되는 액션 버튼은 카드 뷰와 동일 합니다. 한개, 여러개를 선택 했을 때 삭제 버튼이 표시 됩니다.

​

※ 샌드박스 애셋 새로고침​

새로운 샌드박스 애셋을 업로드 하는 경우, 정렬 필터 우측에 새로고침 버튼이 표시됩니다. 새로고침 버튼 클릭 시 현재 애셋 목록을 새로 고침해 다시 표시합니다.

  1. 새로고침 - 현재 애셋 목록을 새로 고침해 다시 표시
  2. Uploading 상태의 샌드박스 애셋을 표시

모델 예측

범위로 선택한 모델을 API를 통해 모델 예측을 할 수 있습니다.

사전 준비​

워크플로우 실행을 하기 위해선 다음과 같은 준비가 필요합니다.

  • 호출을 위한 URL
  • 헤더에 필요한 토큰
  • 호출을 위한 json 형태의 Body

요청 메서드​

  • POST

URL​

  • API를 호출하기 위한 URL은 다음과 같은 형태입니다.
https://호스트주소/api/common/backend/external/api
정보
  • 호스트주소는 서비스되는 도메인 주소 입니다.
  • ex) https://aiip-stage.skcc.com :::

Header​

  • API를 호출하기 위한 Header는 다음과 같은 형태입니다.
Authorization: Bearer 토큰
Content-Type: application/json
  • 토큰 발급 화면으로 이동합니다.
  • 발급된 토큰은 "Bearer "를 붙여 호출하여 사용해야 합니다.
  • ex) Bearer eyJ0eXAiOiJKV1QiLCJhbGciO... :::

Body​

  • API를 호출하기 위한 Body는 다음과 같은 형태입니다.
{  
  "apiId": "RUNTIME_PREDICT"
  "projectId": 프로젝트 아이디
  "modelId": 모델 아이디
  "apiParameter": {
    "instances":  [ [ int, "string", float, ], ],
    "labels": [ "string", ]
  }
}

API 호출을 위해선 다음의 변수가 필수로 요구됩니다.​

변수명설명
apiIdAPI를 호출하기 위한 ID로 "RUNTIME_PREDICT"를 값으로 합니다.
projectIdAPI를 적용할 프로젝트의 ID로 int 값 입니다.
modelIdAPI를 적용할 모델의 ID로 string 값 입니다.
apiParameter.instances모델을 구성하는 한 개 이상 tuple의 배열 입니다.
apiParameter.labels모델을 구성하는 전체 attribite의 string 배열 입니다.

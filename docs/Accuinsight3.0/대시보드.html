  3. 글로벌 애셋 리스트로 이동

  7. 사용자 메뉴 표시

  2. 리스트 보기 모드

4. 실행 현황​


오늘 기준으로 실행 현황을 표시

  1. 필터 - 전체 실행 현황, 내가 오너인 실행 현황
    • 전체 실행 현황 - 전체 실행 현황을 수집
    • 내가 오너인 실행 현황 - "내가 오너인" 실행 현황을 수집
  2. 스케줄 현황 표시
  3. 생성중인 AutoML, Labelling, Clustering 현황 표시
  4. 상태가 Running인 Sandbox (Jupyter, R) 현황 표시

5. 자원관리​


Accuinsight+ 3.0에 등록되어 있는 클러스터 자원 현황 표시

정보

서비스 관리자일 경우에만 표시가 됩니다.

  1. Chevron - 자원현황 표시를 열거나 닫음
  2. 클러스터 필터 - 등록된 클러스터별로 필터 수행
  3. 클러스터 자원 상태 표시
    • 경고 - CPU, Memory, GPU 사용이 경고(90% 이상)인 항목 수
    • 주의 - CPU, Memory, GPU 사용이 주의(70% 이상)인 항목 수
    • 정상 - CPU, Memory, GPU 사용이 정상(70% 미만)인 항목 수
  4. 클러스터의 CPU, Memory, GPU 사용 현황 표시
  5. 네임스페이스 필터 - 클러스터에 생성된 네임스페이스 필터 수행
  6. 네임스페이스 자원 상태 표시
    • 경고 - CPU, Memory, GPU 사용이 경고(90% 이상)인 항목 수
    • 주의 - CPU, Memory, GPU 사용이 주의(70% 이상)인 항목 수
    • 정상 - CPU, Memory, GPU 사용이 정상(70% 미만)인 항목 수

  1. View 권한 이상 포함된 모든 프로젝트 수를 표시

  1. 프로젝트 리스트로 이동
  2. 북마크 설정/해제
정보

북마크 설정 시 우선적으로 정렬되어 리스트 상위에 표시 됩니다.

  2. 프로젝트의 More Actions 메뉴
    • 새 워크플로우 시작 - 워크플로우 생성 화면으로 이동
    • 기본정보 편집 - 프로젝트 수정 화면으로 이동
    • 프로젝트 복제 - 프로젝트 복제 수행
    • 삭제 - 프로젝트 삭제 수행
정보

프로젝트 구성요소(워크플로우, 샌드박스, 모델)를 먼저 삭제해야 삭제가 가능 합니다.

  1. 프로젝트 생성 페이지로 이동

8. 샌드박스​

  1. 모든 프로젝트에 속한 샌드박스 수를 표시

  2. 샌드박스 리스트로 이동

    정보

    필터 선택 시 조건(프로젝트, 워크플로우, 샌드박스 타입)을 통한 리스트 필터링

    • 프로젝트 - 샌드박스가 속한 프로젝트 선택
    • 워크플로우 - 본 계정으로 생성한 워크플로우 또는 워크플로우 없는 샌드박스(stand-alone)
    • 샌드박스 타입 - Jupyter Lab 또는 R Studio 선택

  3. 샌드박스 View 새로고침

  4. 북마크 설정/해제

정보

북마크 설정 시 우선적으로 정렬되어 리스트 상위에 표시 됩니다.

  1. 샌드박스의 More Actions 메뉴

    • 샌드박스가 Stop 상태일 경우
      • 시작 - 샌드박스를 실행시켜 샌드박스를 Running 상태로 변경
      • 삭제 - 샌드박스 삭제 수행
    • 샌드박스가 Waiting 상태일 경우
      • 삭제 - 샌드박스 삭제 수행
    • 샌드박스가 Running 상태일 경우
      • 중지 - 샌드박스 중지
        • 샌드박스 중지 시 해당 샌드박스 이미지 백업 후 시작 시 해당 백업 이미지 사용
      • 샌드박스 애셋 저장 - 실행중인 샌드박스를 에셋으로 저장할 수 있는 팝업 표시
        1. 애셋 이름 - 저장할 샌드박스 애셋의 이름 입력
        2. 소스 포함 여부 - 샌드박스에 포함된 소스를 함께 저장할지 설정
        3. 설명 - 저장할 샌드박스 애셋의 상세 설명 입력
        4. 공유범위 설정 - 저장할 샌드박스 애셋의 공유 범위 설정
        5. 취소 - 샌드박스 애셋 저장을 취소하고 대시보드 메인 화면으로 돌아가기
        6. 저장 - 입력한 내용으로 샌드박스 애셋 저장 진행
      • 삭제 - 샌드박스 삭제 수행

  2. 샌드박스의 상태 및 변경 확인

    • Pending : 샌드박스 환경 생성 대기 중
    • Waiting : 샌드박스 환경 생성 중
    • Running : 샌드박스 환경에 접속 가능한 정상 실행중인 상태
    • Stop : 샌드박스 환경이 중지 상태이며, 접속 불가능.
정보

샌드박스 클릭 시 해당 샌드박스로 접근

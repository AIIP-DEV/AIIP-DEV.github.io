데이터 가져오기

워크플로우에서 사용될 데이터를 불러옵니다.

노드 메뉴에서 [데이터] 카테고리를 선택하고 우측패널의 [데이터 가져오기]를 선택하여 노드를 추가합니다.

메뉴​

  • 데이터 불러오기 전

    • 이름바꾸기 : 데이터노드의 노드명 변경
    • 즐겨찾기에 추가 : 즐겨찾기에 [데이터 가져오기]노드 추가
    • 데이터 가져오기 : 데이터셋 추가 화면으로 이동
    • 삭제 : 선택된 데이터 노드를 워크플로우에서 제거

  • 데이터 불러오기 후

    • 변수 설정 : 데이터 불러오기시 파일 이름을 고정시키지 않고, 변수화하여 불러오기
    • 데이터 보기
      • 전체 화면으로 보기 : 데이터 미리보기 화면으로 이동
      • 팝업으로 보기 : 데이터 미리보기 팝업 호출
    • Wrangler : 선택된 데이터노드에 Wrangler노드를 추가
    • 차트생성 : 차트 생성 화면으로 이동
    • 이름바꾸기 : 데이터노드의 노드명 변경
    • 데이터셋 애셋으로 저장 : 데이터셋을 애셋으로 저장하기
    • 데이터셋 내보내기: 데이터셋 내보내기 화면으로 이동
    • DataRobot에 내보내기 : DataRobot에 내보내기
    • 다운로드: 데이터를 다운로드
    • 데이터셋 비우기 : 데이터노드에서 데이터를 초기화
    • 즐겨찾기에 추가 : 즐겨찾기에 [데이터 가져오기]노드 추가
    • 삭제 : 선택된 데이터 노드를 워크플로우에서 제거

데이터셋 추가​

  1. 로컬 파일 : 로컬에 있는 data파일을 불러옵니다.
  2. Amazon S3 : AWS S3에 저장된 데이터를 불러옵니다.
  3. RDB : RDBMS에 저장된 데이터를 불러옵니다.
  4. HTTP : HTTP 프로토콜을 사용하여 데이터를 불러옵니다.
  5. HIVE : HIVE 데이터 웨어하우스에서 데이터를 불러옵니다.
  6. IMPALA : IMPALA에 저장된 데이터를 불러옵니다.
  7. HDFS : 하둡 파일 시스템에 저장된 데이터를 불러옵니다.
  8. Blob : Azure Storage 에 저장된 데이터를 불러옵니다.
  9. Snowflake : SnowFlake에 저장된 데이터를 불러옵니다.
  10. Asset : 다른 워크플로우에서 저장한 데이터셋 애셋을 불러옵니다.
  11. NAS : NAS 저장소에서 데이터를 불러옵니다.
  • 로컬 파일

  1. 데이터셋 추가 : 파일을 Drag & Drop 하거나 Browes 버튼을 클릭하여 파일을 선택 합니다.
  2. More Actions : 로컬파일 불러오기 설정 팝업 호출
  3. 삭제 : 해당 데이터셋을 삭제
  4. 추가 : 추가대상의 데이터셋을 워크플로우에 로드
  • "로컬파일 불러오기 설정" 팝업

  1. format : 불러올 파일의 형식 지정(csv, json, xml, text, parquet, orc 중 택1)

  2. Delimiter : 구분자 입력

  3. Option : 사용 가능한 옵션설정(헤더유무, encoding, nanValue, nullValue) 설정

    • key: header, value : true/False = 헤더 설정 여부(True : 헤더설정, False : 헤더미설정)
    • key: encoding, value : 인코딩값 = 인코딩 설정
    • key: nullValue, value : 입력값(ex:NULL) = 데이터에 포함된 입력값(NULL)을 Null문자열 처리
    • key: nanValue, value : 입력값(ex:NaN) = 데이터에 포함된 입력값(NaN)을 NaN문자열 처리

  4. Option 추가 : 옵션이 여러개일때 항목을 추가

  5. Option 삭제 : 선택된 옵션 삭제

  6. Custom Schema : 컬럼에 대한 데이터타입 설정

    • column: 데이터타입 변경하려는 컬럼 명, value : 데이터타입 설정 (String, Boolean, Decimal, Double, Float, Integer, Long, Short, Timestamp)

  7. Column Option 추가 : 컬럼 옵션이 여러개일때 항목을 추가

    • 설정을 추가할 경우 추가된 컬럼 정보로 데이터셋 가져오기가 실행되며, 설정을 추가하지 않는 경우 모든 컬럼 정보로 데이터셋 가져오기가 실행됨
    • 데이터셋 가져오기 완료후 컬럼 설정 변경은 데이터셋 노드의 우패널 부가 옵션 "설정"을 통해 추가/삭제가 가능함.

  8. Column Option 삭제 : 선택된 컬럼 옵션 삭제

  9. 취소 : 입력된 내용을 적용하지 않고 설정 팝업을 닫습니다.

  10. 반영하여 추가 : 입력된 내용을 적용 합니다.

  1. header : HTTP 요청헤더 (항목이 여러개일때 [+]버튼으로 추가)

  2. format : 불러올 파일의 데이터 포맷 선택(json, xml중 택1)

  3. Delimiter : 불러올 데이터의 파싱 구분자 입력

  4. Option : 사용 가능한 옵션설정(헤더유무, nanValue, nullValue) 설정 (항목이 여러개일때 [+]버튼으로 추가)

    • key: header, value : true/False = 헤더 설정 여부(True : 헤더설정, False : 헤더미설정)

  5. Params : 노드에서 사용할 parameter 입력 (항목이 여러개일때 [+]버튼으로 추가)

  6. Custom Schema : column 및 dataType 정의 (항목이 여러개일때 [+]버튼으로 추가)

    • 설정을 추가할 경우 추가된 컬럼 정보로 데이터셋 가져오기가 실행되며, 설정을 추가하지 않는 경우 모든 컬럼 정보로 데이터셋 가져오기가 실행됨
    • 데이터셋 가져오기 완료후 컬럼 설정 변경은 데이터셋 노드의 우패널 부가 옵션 "설정"을 통해 추가/삭제가 가능함.

  7. 취소 : 입력된 내용을 적용하지 않고 설정 팝업을 닫습니다.

  5. 추가 : 추가대상의 데이터셋을 워크플로우에 로드
  • "NAS 불러오기 설정" 팝업

  1. format : 불러올 파일의 형식 지정(csv, json, xml, text, parquet, orc 중 택1)

  2. Delimiter : 구분자 입력

  3. Option : 사용 가능한 옵션설정(헤더유무, nanValue, nullValue) 설정

    • key: header, value : true/False = 헤더 설정 여부(True : 헤더설정, False : 헤더미설정)
    • key: encoding, value : 인코딩값 = 인코딩 설정
    • key: nullValue, value : 입력값(ex:NULL) = 데이터에 포함된 입력값(NULL)을 Null문자열 처리
    • key: nanValue, value : 입력값(ex:NaN) = 데이터에 포함된 입력값(NaN)을 NaN문자열 처리

  4. Option 추가 : 옵션이 여러개일때 항목을 추가

  5. Option 삭제 : 선택된 옵션 삭제

  6. Custom Schema : 컬럼에 대한 데이터타입 설정

    • column: 데이터타입 변경하려는 컬럼 명, value : 데이터타입 설정 (String, Boolean, Decimal, Double, Float, Integer, Long, Short, Timestamp)

  7. Column Option 추가 : 컬럼 옵션이 여러개일때 항목을 추가

    • 설정을 추가할 경우 추가된 컬럼 정보로 데이터셋 가져오기가 실행되며, 설정을 추가하지 않는 경우 모든 컬럼 정보로 데이터셋 가져오기가 실행됨
    • 데이터셋 가져오기 완료후 컬럼 설정 변경은 데이터셋 노드의 우패널 부가 옵션 "설정"을 통해 추가/삭제가 가능함.

  8. Column Option 삭제 : 선택된 컬럼 옵션 삭제

  9. 취소 : 입력된 내용을 적용하지 않고 설정 팝업을 닫습니다.

  10. 반영하여 추가 : 입력된 내용을 적용 합니다.

    • 닫기 : 선택된 컬럼만 표시

    • 적용 : 선택된 필터 적용

  4. 차트 생성하기 : 입력된 내용으로 차트 생성

데이터셋 애셋으로 저장​

선택된 데이터노드를 데이터셋 애셋으로 저장합니다.

  1. 데이터셋 애셋 이름 : 저장할 데이터셋 애셋 이름
  2. 설명 : 데이터셋 애셋의 설명
  3. 공개범위 설정 : 데이터셋 애셋의 공개범위 설정
    • 나만보기 : 로그인한 계정에만 공개
    • 프로젝트 내 공개 : 현재 프로젝트에 공개
    • 글로벌 : 모든 사용자에게 공개
  4. 취소 : 저장하지 않고 팝업 종료
  5. 등록 : 입력된 내용으로 데이터셋 애셋을 생성

데이터셋 내보내기​

선택된 데이터노드를 내보내기

  • 우클릭으로 데이터셋 내보내기, 내보내기 수정, 내보내기 삭제를 진행할 수 있습니다.

내보내기 수정​

  • 선택한 데이터 노드의 내보내기 리스트 확인이 가능하며, 파일명, 확장자, 규칙 등을 수정할 수 있습니다.
  • livy를 통해 이미 내보내진 파일은 수정되지 않습니다.

내보내기 삭제​

  • 선택한 데이터 노드의 내보내기 리스트 확인이 가능하며, 다중으로 선택하여 내보내기 설정을 삭제할 수 있습니다.
  • livy를 통해 이미 내보내진 파일은 삭제되지 않습니다.

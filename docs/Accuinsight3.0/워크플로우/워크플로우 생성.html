워크플로우 생성

워크플로우 생성​

  • 프로젝트에서 워크플로우 생성을 클릭합니다.

워크플로우 타입​

  • 워크플로우 타입과 템플릿을 선택후 다음을 클릭합니다.
  1. Pyspark 워크플로우
  2. Snowflake 워크플로우
  3. Python 워크플로우

템플릿​

  1. 빈 워크플로우
  2. AutoML 바로가기
  3. Sandbox로 바로가기
  4. Clustering으로 바로가기

1. 빈 워크플로우​

템플릿 중 '빈 워크플로우'를 선택하여 데이터셋 추가 화면으로 이동합니다.

  • 데이터셋 추가 방법
    • 로컬 파일 - 로컬에 있는 파일을 Drag & Drop 혹은 Browse 버튼을 선택하여 추가
    • Amazon S3 - 등록되어 있는 S3버킷에서 선택하여 데이터셋 추가
    • RDB - 등록되어 있는 RDBMS에서 선택하여 데이터셋 추가
    • HTTP - HTTP URL로 데이터셋을 추가
    • HIVE - 등록되어 있는 하둡 클러스터에서 HIVE 계정을 선택하여 데이터셋 추가
    • IMPALA - 등록되어 있는 하둡 클러스터에서 IMPALA 계정을 선택하여 데이터셋 추가
    • HDFS - 등록되어 있는 하둡 클러스터에서 데이터셋 추가
    • Blob - 등록되어 있는 Blob 컨테이너에서 선택하여 데이터셋 추가
    • SnowFlake - 등록되어 있는 SnowFlake에서 선택하여 데이터셋 추가
    • Asset - 등록되어 있는 데이터셋 애셋을 선택하여 데이터셋 추가
    • NAS - 등록되어 있는 NAS 디렉토리 선택하여 데이터셋 추가

  • 데이터셋 추가 방법에 대한 상세내용은 데이터를 참고하십시오.

    데이터셋을 추가하여 워크플로우 생성합니다.

  • 건너뛰기 - 데이터셋을 추가하지 않고 워크플로우 화면으로 넘어갑니다.

2. Sandbox로 바로가기​

  1. 템플릿 중 'Sandbox로 바로가기'를 선택하여 Sandbox분석 환경 설정 화면으로 이동합니다.
  2. Sandbox분석 환경 설정 후 분석 환경 생성하기 진행합니다.
  3. 워크플로우에 Sandbox 노드가 생성됩니다.
  • Sandbox분석 환경 상세 설정 방법은 Sandbox를 참고하십시오.

3. AutoML로 바로가기​

  1. 템플릿 중 'AutoML로 바로가기'를 선택하여 데이터셋 추가 화면으로 이동합니다.
  2. 데이터셋을 추가 합니다.
  3. 추가한 데이터셋에 랭글러가 추가되며, 랭글러에서 데이터 처리를(처리할 내용이 없으면 건너뛰기로)진행 합니다.
  4. AutoML 노드가 생성됩니다.
  • 데이터를 추가해야 AutoML 노드 생성이 동작합니다.

4. Labelling으로 바로가기​

  1. 템플릿 중 'Labelling으로 바로가기'를 선택하여 데이터셋 추가 화면으로 이동합니다.
  2. 데이터셋을 추가 합니다.
  3. 추가한 데이터셋에 랭글러가 추가되며, 랭글러에서 데이터 처리를(처리할 내용이 없으면 건너뛰기로) 진행 합니다.
  4. labelling 노드가 생성됩니다.
  • 데이터를 추가해야 Labelling 노드 생성이 동작합니다.

워크플로우 노드 추가​

  1. 데이터 노드를 클릭하고 노드 메뉴에서 추가할 노드 그룹을 선택하면 해당 노드 타입의 빈 노드가 생성됩니다.
  2. 우측 패널에서 노드를 선택하면 추가 노드가 결정됩니다.
  3. 우측 패널에서 추가된 노드의 항목들을 설정하고 추가(또는 실행)를 선택합니다.
  4. 추가한 노드의 작업이 실행되고 작업이 완료되면 노드 추가가 완료됩니다.
  5. 저장 버튼을 클릭하여 워크플로우를 저장할 수 있습니다.
  • 노드별 상세 설정은 워크플로우 화면을 참고하십시오.

데이터 가져오기

워크플로우에서 사용될 데이터를 불러옵니다.

노드 메뉴에서 [데이터] 카테고리를 선택하고 우측패널의 [데이터 가져오기]를 선택하여 노드를 추가합니다.

메뉴​

  • 데이터 불러오기 전

    • 이름바꾸기 : 데이터노드의 노드명 변경
    • 즐겨찾기에 추가 : 즐겨찾기에 [데이터 가져오기]노드 추가
    • 삭제 : 선택된 데이터 노드를 워크플로우에서 제거

  • 데이터 불러오기 후

    • 데이터 보기
      • 전체 화면으로 보기 : 데이터 미리보기 화면으로 이동
      • 팝업으로 보기 : 데이터 미리보기 팝업 호출
    • 이름바꾸기 : 데이터노드의 노드명 변경
    • 데이터셋 애셋으로 저장 : 선택된 데이터노드를 데이터셋 애셋에 추가
    • 데이터셋 내보내기 : 데이터셋 내보내기 화면으로 이동
    • 다운로드 : 데이터셋 다운로드
    • 데이터셋 비우기 : 데이터노드에서 데이터를 초기화
    • 즐겨찾기에 추가 : 즐겨찾기에 [데이터 가져오기]노드 추가
    • 삭제 : 선택된 데이터 노드를 워크플로우에서 제거

데이터셋 추가​

  1. 로컬 파일 : 로컬에 있는 data파일을 불러옵니다.
  2. NAS : NAS에 저장된 데이터를 불러옵니다.
  3. Amazon S3 : AWS S3에 저장된 데이터를 불러옵니다.
  4. Snowflake : SnowFlake에 저장된 데이터를 불러옵니다.
  • 로컬 파일

  1. 데이터셋 추가 : 파일을 Drag & Drop 하거나 Browes 버튼을 클릭하여 파일을 선택 합니다.
  2. More Actions : 로컬파일 불러오기 설정 팝업 호출
  3. 삭제 : 해당 데이터셋을 삭제
  4. 추가 : 추가대상의 데이터셋을 워크플로우에 로드
  • 로컬파일 불러오기 설정

  1. format : 불러올 파일의 형식 지정(parquet, orc 중 택1)
  2. Delimiter : 구분자 입력
  3. Option : 사용 가능한 옵션설정(헤더유무, nanValue, nullValue) 설정
    • key: header, value : true/False = 헤더 설정 여부(True : 헤더설정, False : 헤더미설정)
    • key: encoding, value : 인코딩값 = 인코딩 설정
    • key: nullValue, value : 입력값(ex:NULL) = 데이터에 포함된 입력값(NULL)을 Null문자열 처리
    • key: nanValue, value : 입력값(ex:NaN) = 데이터에 포함된 입력값(NaN)을 NaN문자열 처리
  4. Option 추가 : 옵션이 여러개일때 항목을 추가
  5. Option 삭제 : 선택된 옵션 삭제
  6. 취소 : 입력된 내용을 적용하지 않고 설정 팝업을 닫습니다.
  7. 반영하여 추가 : 입력된 내용을 적용 합니다.
  • 워크플로우에 로드후 임시테이블에 저장

  1. db : 임시테이블을 저장할 DB 선택
  2. schema : 임시테이블을 저장할 DB의 논리그룹 선택
  3. warehouse : 임시테이블을 저장할 snowflake의 웨어하우스
  4. bucketName : 업로드 파일이 위치한 버킷명 (자동입력)
  5. projectId : 프로젝트의 아이디 (자동입력)
  6. nullif : 입력된 값을 null로 치환
  7. 추가 : 입력된 내용으로 임시테이블 저장
  • NAS

  4. 삭제 : 해당 데이터셋을 삭제

  1. 버킷 선택 : S3 버킷 관리에 등록되어 있는 버킷을 선택하여 파일을 확인
  2. S3 버킷 관리 : [S3 버킷 추가 / 관리] 팝업 호출

  1. 버킷이름 : 추가할 버킷의 이름

  • SQL 쿼리작성

  1. warehouse : 사용할 Snowflake의 웨어하우스
  2. DB 선택 : 불러올 dababase를 선택
  3. 검색 : database내의 table검색
  4. 테이블 선택 : 선택된 DB의 테이블을 클릭 (data Preview생성 및 schema 정보를 불러옵니다.)
    • 더블클릭 : SQL Query입력창에 선택된 테이블을 자동입력
  5. SQL Query : SQL문을 직접 입력 하거나 왼쪽의 테이블, 컬럼을 더블클릭하여 입력 또는 History 리스트에서 더블클릭
  6. History : 실행했던 SQL 쿼리문을 조회 (SQL List 팝업 호출)
  7. validation check : 작성된 SQL문의 유효성 검사
  8. executeQuery : 입력된 SQL문을 실행하여 data Preview생성
  9. Data Preview : 100건의 데이터를 조회
  10. Schema : 왼쪽 테이블리스트에서 선택된 테이블의 schema정보를 표기
  11. 취소 : SQL 쿼리작성 팝업을 닫습니다.
  12. 적용 : 작성된 SQL문을 적용하여 데이터셋을 추가

데이터 보기​

Data​

데이터 미리보기

  1. 뒤로가기 : 이전페이지로 이동

  2. 팝업으로 보기 : 현재 페이지를 팝업으로 호출

  3. More Actions : 데이터 노드 메뉴와 동일

    • 다운로드 : 현재 preview 데이터를 로컬에 다운로드

  4. 닫기 : 우측패널 닫기

  5. 패널 최소화 : 우측패널 열기 / 닫기

  6. 컬럼 필터

    • 검색 : 컬럼명 검색
    • 선택 : 화면에 표시할 컬럼명 멀티선택
    • 취소 : 적용하지 않고 닫기
    • 닫기 : 선택된 컬럼만 표시

  7. preview 컬럼 set 선택 (10개 단위로 선택)

  8. 이전 10개 컬럼 set으로 변경 / 다음 10개 컬럼 set으로 변경

데이터셋 애셋으로 저장​

선택된 데이터노드를 데이터셋 애셋으로 저장합니다.

  1. 데이터셋 애셋 이름 : 저장할 데이터셋 애셋 이름
  2. 설명 : 데이터셋 애셋의 설명
  3. 공개범위 설정 : 데이터셋 애셋의 공개범위 설정
    • 나만보기 : 로그인한 계정에만 공개
    • 프로젝트 내 공개 : 현재 프로젝트에 공개
    • 글로벌 : 모든 사용자에게 공개
  4. 적용된 노드 : 선택된 데이터노드를 만들기위해 적용되었던 Acction 노드들
  5. 취소 : 저장하지 않고 팝업 종료
  6. 등록 : 입력된 내용으로 데이터셋 애셋을 생성

내보내기​

선택된 데이터노드를 내보내기

NAS​

  1. 등록된 NAS 디렉터리를 선택하여 내보내기
  2. 폴더 추가 : 선택된 폴더에 새폴더 생성
  3. 추가 : 선택한 폴더 및 파일에 내보내기
  4. 내보내기 경로 : 내보내기할 NAS 경로 (자동입력)
  5. 파일명 : 내보내기할 파일명
  6. 파일 확장자 : 내보낼 파일의 형식 지정(csv, json, txt, parquet, orc 중 택1)
  7. 내보내기 규칙 : 같은 이름의 파일이 존재할때 형식 지정
    • 이어 붙임
    • 덮어 씀
    • 저장하지 않고 건너 뜀
    • 에러 처리
  8. 우측 패널에 입력된 내용으로 내보내기

AWS S3​

  1. 버킷 선택 : S3 버킷 관리에 등록되어 있는 버킷을 선택하여 내보내기
  2. S3 버킷 관리 : [S3 버킷 추가 / 관리] 팝업 호출
  3. 폴더 추가 : 선택된 폴더에 새폴더 생성
  4. 추가 : 선택한 폴더 및 파일에 내보내기
  5. 내보내기 경로 : 내보내기할 S3 경로 (자동입력)
  6. 파일명 : 내보내기할 파일명
  7. 파일 확장자 : 내보낼 파일의 형식 지정(csv, json, txt, parquet, orc 중 택1)
  8. 내보내기 규칙 : 같은 이름의 파일이 존재할때 형식 지정

모델 예측 API

  • 배포한 모델로 예측 요청하기 위한 API 사용 방법을 설명합니다.
  • Tabular 모델과 Image 모델 예제를 제공합니다.

사전 준비​

​

1) Tabular 모델​

워크플로우 실행을 하기 위해선 다음과 같은 준비가 필요합니다.

  • API 키 관리에서 발급한 apikeyId / apikey / 토큰
  • 호출을 위한 URL
  • request 정보 (method / header / body)
​

(1) API 키 관리에서 발급한 apikeyId / apikey / 토큰​

  • API 키 관리 가이드를 참고하여 발급받을 수 있습니다.
  • 발급 시, API 호출을 수행할 모델을 등록하여, 권한을 부여해야 합니다.
​

(2) 호출을 위한 URL​

  • API를 호출하기 위한 URL은 모델 레지스트리 > 서비스 대시보드 > 기본 정보 설정에서 확인 가능하며 다음과 같은 형태입니다.
https://호스트주소/api/runtime/ifservice/predict/모델ID
정보
  • 호스트주소는 서비스되는 도메인 주소 입니다.
  • ex) https://aiip.skcc.com
​

(3) request 정보 (method / header / body)​

method​
  • post
Header​
  • API를 호출하기 위한 Header는 다음과 같은 형태입니다.
Api-Auth-Token: API 키 관리에서 발급한 토큰
Content-Type: application/json
Body​
  • API를 호출하기 위한 Body는 다음과 같은 application/json 형태입니다.
{  
    "instances":  [ [ float, float, float, float, "String" ], ... ],
    "labels": [ "String","String","String","String","String" ]
}

모델 예측하기​

  • API를 통한 모델 예측은 2가지 방법이 있습니다.
  • 예측 환경에 따라 적절하게 사용할 수 있습니다.
​

방법 1: API 호출을 통한 예측​

  • 사전 준비를 통해 부여받은 URL / 토큰 정보를 토대로 API 예측 요청을 합니다.

Tabular 모델 예측 예시​

URL​

https://aiip.skcc.com/api/runtime/ifservice/predict/mdl-e24f7f6d-8264-44fa-b15e-175f27eefb7b

Header​

Api-Auth-Token: eyJ0eXAiOiJKV1QiLCJhbGci...Iq9Q5crNxn1DxA

Body​

{
    "instances": [
        [
            4.7,
            3.2,
            1.3,
            0.2
        ],
        [
            4.6,
            3.1,
            1.5,
            0.2
        ]
    ],
    "labels": [
        "sepal_length",
        "sepal_width",
        "petal_length",
        "petal_width"
    ]
}

호출 결과​

{
    "result": {
        "model_version": "2",
        "predictions": [
            "Setosa",
            "Setosa"
        ]
},
    "statusCode": 0,
    "httpStatusCode": 200
}

방법 2: 코드 실행을 통한 예측​

  • 사전 준비를 통해 부여받은 URL / apikeyId / apikey 를 토대로 코드를 실행하여 예측 요청을 합니다.
  • 샘플 소스 코드는 Java / Python 두 종류로 제공됩니다.

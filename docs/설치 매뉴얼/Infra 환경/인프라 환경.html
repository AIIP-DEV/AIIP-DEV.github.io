Infra 환경

Accuinsight 3.0을 설치하기 위한 K8S 환경 설정

Product EKS 환경 구성​

사전 구성​

  1. API User 생성

    1. AWS Console → IAM → Users → Add users
    2. user name 입력 (ex: api-eks-admin)
    3. Access Type → Programmatic Access 만 Check (Console 접속 없는 API 전용 계정)
    4. Permission → AdministratorAccess Policy 적용

  2. Key 생성

    aws kms create-alias --alias-name alias/eksworkshop --target-key-id $(aws kms create-key --query KeyMetadata.Arn --output text)

    export MASTER_ARN=$(aws kms describe-key --key-id alias/eksworkshop --query KeyMetadata.Arn --output text)

EKS 구성​

  1. EKS Cluster 생성

    eksctl create cluster -f aiip-prod-eks.yaml

    • aiip-prod-eks.yaml
      ---
      apiVersion: eksctl.io/v1alpha5
      kind: ClusterConfig

      metadata:
        name: aiip-prod
        region: ap-northeast-2
        version: "1.18"

      availabilityZones: ["ap-northeast-2a", "ap-northeast-2b", "ap-northeast-2c"]

      managedNodeGroups:
      - name: master
        desiredCapacity: 2
        instanceType: m5a.4xlarge
        ssh:
          enableSsm: true
      - name: worker
        desiredCapacity: 3
        instanceType: m5a.4xlarge
        ssh:
          enableSsm: true
      - name: runtime
        desiredCapacity: 1
        instanceType: m5a.12xlarge
        ssh:
          enableSsm: true
      - name: gpu
        desiredCapacity: 1
        instanceType: p2.8xlarge
        ssh:
          enableSsm: true

      # To enable all of the control plane logs, uncomment below:
      # cloudWatch:
      #  clusterLogging:
      #    enableTypes: ["*"]

      secretsEncryption:
        keyARN: ${MASTER_ARN}

    현재 yaml에는 vpc가 default(192.168.0.1/16)으로 생성되며 변경시 아래 항목 추가

    vpc:
    cidr: 10.190.0.0/16
    clusterEndpoints:
    publicAccess: true
    privateAccess: true

    https://www.eksworkshop.com/030_eksctl/launcheks/

  2. Kube configmap update

    eksctl create iamidentitymapping --cluster aiip-prod --arn ${user of role arn} --group system:masters --username admin

  3. ingress controller 구성

    1. OIDC Provider 연결

      export CLUSTER_NAME=aiip-prod
      export AWS_REGION=ap-northeast-2
      eksctl utils associate-iam-oidc-provider --region=$AWS_REGION --cluster $CLUSTER_NAME --approve

    2. Policy 생성

      curl -o iam-policy.json https://raw.githubusercontent.com/kubernetes-sigs/aws-load-balancer-controller/main/docs/install/iam_policy.json

      aws iam create-policy \
          --policy-name AWSLoadBalancerControllerIAMPolicy \
          --policy-document file://iam-policy.json

      aiip-prod 환경에서는 console에서 추가하였으며, POLICY-EKS-IAM으로 생성되어있음

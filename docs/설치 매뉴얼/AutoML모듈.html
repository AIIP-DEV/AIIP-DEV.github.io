AutoML모듈

Accuinsight 3.0 AutoML모듈 설치 방법

Requirements​

  • AccuTuning image를 받을 수 있는 ECR 경로
    • 본 메뉴얼에서는 harbor.accuinsight.net에 구성된 경로를 사용하였습니다.
    • ECR이 변경되면 아래 yml 파일 중 해당 부분을 변경하시면 됩니다.
  • 필요한 yml 파일
    • 아래에 텍스트로 직접 붙였습니다.
    • 각 파일 이름에 맞게 만드시면 됩니다.
  • aws configure 정보
    • k8s 담당자에게 문의하세요.

AWS Configure​

  • 이 내용은 본 메뉴얼에서 다루지 않습니다. 
    aws configure

Service Account 설정하기​

  • 아래 내용을 복사하여 base.yml을 만들고 apply 하세요. (pod를 제어하려면 ServiceAccount, Role, Clusterrole 설정이 필요합니다)

    kubectl apply -f base.yml
    apiVersion: v1
    kind: ServiceAccount
    metadata:
    name: accutuning
    namespace: aiip-tuning
    ---
    apiVersion: rbac.authorization.k8s.io/v1beta1
    kind: Role
    metadata:
    name: accutuning
    namespace: aiip-tuning
    rules:
    - apiGroups: ["*"]
    resources: ["pods"]
    verbs: ["get", "list", "watch", "create", "update", "patch", "delete"]
    ---
    apiVersion: rbac.authorization.k8s.io/v1beta1
    kind: RoleBinding
    metadata:
    name: accutuning
    roleRef:
    kind: Role
    name: accutuning
    apiGroup: rbac.authorization.k8s.io
    subjects:
    - kind: ServiceAccount
    name: accutuning
    namespace: aiip-tuning
    - kind: Group
    name: system:authenticated
    apiGroup: rbac.authorization.k8s.io
    - kind: Group
    name: system:unauthenticated
    apiGroup: rbac.authorization.k8s.io
    ---
    apiVersion: rbac.authorization.k8s.io/v1
    kind: ClusterRole
    metadata:
    name: accutuning-cluster-role
    rules:
    - apiGroups: ["*"]
    resources: ["*"]
    verbs: ["*"]
    ---
    apiVersion: rbac.authorization.k8s.io/v1
    kind: ClusterRoleBinding
    metadata:
    name: accutuning-cluster-rolebinding
    roleRef:
    kind: ClusterRole
    name: accutuning-cluster-role
    apiGroup: rbac.authorization.k8s.io  
    subjects:
    - kind: ServiceAccount
    name: accutuning
    namespace: aiip-tuning

PV, PVC 설정하기​

  • 아래 내용으로 pv.yml, pvc.yml을 만들고 apply 하세요.

  • 파일 안에 volumeHandle은 k8s 담당자에게 별도 문의하세요. 

    kubectl apply -f pv.yml
    apiVersion: v1
    kind: PersistentVolume
    metadata:
    name: efs-pv
    spec:
    capacity:
        storage: 5Gi
    volumeMode: Filesystem
    accessModes:
        - ReadWriteMany
    persistentVolumeReclaimPolicy: Retain
    storageClassName: efs-sc
    csi:
        driver: efs.csi.aws.com
        volumeHandle: fs-00000000
    kubectl apply -f pvc.yml
    apiVersion: v1
    kind: PersistentVolumeClaim
    metadata:
    name: efs-claim
    namespace: aiip-tuning
    spec:
    accessModes:
        - ReadWriteMany
    storageClassName: efs-sc
    resources:
        requests:
        storage: 5Gi

각종 app 관련 설정​

  • rabbitmq 설정

    • 아래 내용을 app-rabbitmq.yml 파일로 만들고 apply 하세요. 

      kubectl -f apply app-rabbitmq.yml
      apiVersion: apps/v1
      kind: Deployment
      metadata:
      name: accutuning-rabbitmq
      namespace: aiip-tuning
      labels:
          app: accutuning-rabbitmq
      spec:
      replicas: 1
      selector:
          matchLabels:
          app: accutuning-rabbitmq
          tier: accutuning-rabbitmq
      template:
          metadata:
          labels:
              app: accutuning-rabbitmq
              tier: accutuning-rabbitmq
          spec:
          serviceAccount: accutuning
          serviceAccountName: accutuning
          containers:
              - name: rabbitmq
              image: rabbitmq:3.7.8
              ports:
                  - containerPort: 5672
          nodeSelector:
              nodetype: master
          imagePullSecrets:
              - name: regcred
      ---
      apiVersion: v1
      kind: Service
      metadata:
      labels:
          app: accutuning-rabbitmq
      name: accutuning-rabbitmq-svc
      namespace: aiip-tuning
      spec:
      ports:
      - port: 5672
          protocol: TCP
          targetPort: 5672
      selector:
          app: accutuning-rabbitmq
          tier: accutuning-rabbitmq
      type: ClusterIP

공통모듈

Accuinsight 3.0 공통모듈 설치 방법

Requirements​

  • aws configure 정보
    • k8s 담당자에게 문의하세요.
  • aiip-integrated image를 받을 수 있는 ECR 경로
    • 본 메뉴얼에서는 harbor.accuinsight.net에 구성된 경로를 사용하였습니다.
    • ECR이 변경되면 아래 yml 파일 중 해당 부분을 변경하시면 됩니다.
  • 필요한 yml 파일
    • 아래에 텍스트로 직접 붙였습니다.
    • 각 파일 이름에 맞게 만드시면 됩니다.

Namespace 생성​

  • 아래 내용을 복사하여 prod-namespace.yaml을 만들고 apply 하세요.
    kubectl apply -f prod-namespace.yaml
    apiVersion: v1
    kind: Namespace
    metadata:
      name: aiip-common

DATABASE 생성​

  • DB에 접속하여 아래 DDL을 실행하여 DATABASE를 생성 합니다.

    CREATE DATABASE `aiip_integrated_db` /*!40100 DEFAULT CHARACTER SET utf8 */;

    CREATE TABLE `TBL_PROJECT` (
      `ID` bigint(20) NOT NULL AUTO_INCREMENT,
      `NAME` varchar(50) NOT NULL,
      `DESCRIPTION` varchar(500) DEFAULT NULL,
      `THUMB_TYPE` varchar(10) NOT NULL COMMENT 'BASE64, EMOJI',
      `THUMB` text NOT NULL COMMENT 'BASE64 IMAGE OR EMOJI UNICODE',
      `CLUSTER_ID` bigint(20) NOT NULL,
      `RESOURCE_TYPE` varchar(10) NOT NULL,
      `CPU_QUOTA` float NOT NULL,
      `MEM_QUOTA` float NOT NULL,
      `GPU_QUOTA` float NOT NULL,
      `REPLICA_QUOTA` int(11) NOT NULL,
      `LIVY_ID` varchar(50) DEFAULT NULL COMMENT '데이터 타입은 차후 확인 후 변경 필요',
      `LIVY_ALLOCATED_AT` datetime NOT NULL DEFAULT current_timestamp() COMMENT 'LIVY 세션 할당일시',
      `CREATOR` varchar(50) NOT NULL,
      `MODIFIER` varchar(50) NOT NULL,
      `MODIFIED_AT` datetime NOT NULL DEFAULT current_timestamp() ON UPDATE current_timestamp(),
      `CREATED_AT` datetime NOT NULL DEFAULT current_timestamp(),
      PRIMARY KEY (`ID`),
      UNIQUE KEY `UK_PROJECT_NAME` (`ID`,`NAME`)
    );

    CREATE TABLE `TBL_ROLE` (
      `ID` int(11) NOT NULL AUTO_INCREMENT,
      `NAME` varchar(50) NOT NULL,
      `TITLE` varchar(50) NOT NULL,
      `ICON_NAME` varchar(50) NOT NULL,
      `DESCRIPTION` varchar(500) DEFAULT NULL,
      `SCOPE_PROJECT` varchar(5) NOT NULL COMMENT '프로젝트 권한 CRUD : C-생성, R-읽기, U-수정, D-삭제',
      `SCOPE_RESOURCE` varchar(5) NOT NULL COMMENT '프로젝트 리소스 권한 CRUD : C-생성, R-읽기, U-수정, D-삭제',
      `SCOPE_WORKFLOW` varchar(5) NOT NULL COMMENT '프로젝트 워크플로우 권한 CRUD : C-생성, R-읽기, U-수정, D-삭제',
      `SCOPE_USER` varchar(5) NOT NULL COMMENT '프로젝트 사용자 권한 CRUD : C-생성, R-읽기, U-수정, D-삭제',
      `CREATOR` varchar(50) NOT NULL DEFAULT 'SYSTEM',
      `MODIFIER` varchar(50) NOT NULL DEFAULT 'SYSTEM',
      `MODIFIED_AT` datetime NOT NULL DEFAULT current_timestamp() ON UPDATE current_timestamp(),
      `CREATED_AT` datetime NOT NULL DEFAULT current_timestamp(),
      PRIMARY KEY (`ID`)

    -- Create DB
    CREATE DATABASE `bitool` /*!40100 DEFAULT CHARACTER SET utf8 */;

    -- 사용자 등록 및 권한 추가
    CREATE USER bi;
    grant SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, REFERENCES, INDEX, ALTER, LOCK TABLES, EXECUTE, CREATE VIEW, SHOW VIEW, CREATE ROUTINE, ALTER ROUTINE, TRIGGER on *.* to bi with grant option;
    set password for 'bi'@'%' = password('bi1234');
    정보

    BI Tool DB Table의 경우 [aiip-integrated-backend] 서비스가 기동되면서 초기 셋팅인 경우 자동으로 생성합니다.
    [aiip-integrated-backend] 기동후에 반드시 [bitool] 관련 테이블이 생성되었는지 체크 필요.

Service​

  • 아래 내용을 복사하여 k8s-service-prod-be.yaml, k8s-service-prod-ui.yaml을 만들고 apply 하세요.

    • ClusterIP 사용

      kubectl apply -f k8s-service-prod-be.yaml
      kind: Service
      apiVersion: v1
      metadata:
        name: aiip-integrated-backend
        namespace: aiip-common
        labels:
          app: aiip-integrated-backend
      spec:
        ports:
          - name: http
            port: 80
            targetPort: 8080
        selector:
          app: aiip-integrated-backend
        type: ClusterIP
      kubectl apply -f k8s-service-prod-ui.yaml
      kind: Service
      apiVersion: v1
      metadata:
        name: aiip-integrated-ui
        namespace: aiip-common
        labels:
          app: aiip-integrated-ui
      spec:
        ports:
          - name: http
            port: 80
            targetPort: 80
        selector:
          app: aiip-integrated-ui
        type: ClusterIP
      정보

      ServiceTypes으로 LoadBalancer를 사용 하여 ELB 주소(External IP)를 생성하면 해당 ELB 주소로 service 접속이 가능 합니다.

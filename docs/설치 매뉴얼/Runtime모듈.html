Runtime모듈

Accuinsight 3.0 Runtime모듈 설치 방법

Requirements​

  • Kubernetes 클러스터 + cluster-admin 권한을 가지는 계정
  • Elasticsearch 서버
  • MariaDB 서버
  • Ceph 서버
  • Harbor 서버
  • NFS 서버
  • kubectl
  • helm3
  • 설치 파일(사전 제공): release-x.x.zip

이 때, Kubernetes 클러스터의 cluster-admin 권한은 네임스페이스 등의 자원 생성에 필요하다.

설치​

설치파일 준비​

설치파일 압축을 제거한 후, {workingDir}/hack폴더로 진입한다.

cd hack

기본 폴더 구조는 아래와 같다.

./hack
├── apps
├── database
├── fluent-bit.yaml
├── install.env
├── kfserving-installer
└── runtime-installer

환경변수 설정: install.env​

설치를 위해, install.env 파일을 수정한다.


# MariaDB
DB_HOST="db.airuntime.com"
DB_PORT="3306"
DB_USERNAME="admin"
DB_PASSWORD="admin"

# Elasticsearch
ES_HOST="es.airuntime.com"
ES_PORT="443"
ES_USERNAME="admin"
ES_PASSWORD="admin"

# Ceph
CEPH_HOST="s3.airuntime.com"
CEPH_PORT="80"

# Harbor
HARBOR_HOST="harbor.airuntime.com"
HARBOR_PORT="443"

# AIIP
AIIP_HOST="http://aiip-dev.skcc.com"
AIIP_PORT="80"

Persistent Volumes & VolumeMounts 수정​

NFS 서버와 연결하기 위해, 아래 항목을 참고하여 PV & PVC를 설정한다. 아래 예시는 AWS EFS를 사용하는 예시이다.

  • ./apps/resources/external/pv.yaml:
apiVersion: v1
kind: PersistentVolume
metadata:
  name: aiip-runtime-pv
spec:
  accessModes:
  - ReadWriteMany
  capacity:
    storage: 40Gi
  claimRef:
    apiVersion: v1
    kind: PersistentVolumeClaim
    name: aiip-runtime-pvc
    namespace: aiip-runtime
  csi:
    driver: efs.csi.aws.com
    volumeHandle: fsap-07f23a2ea21fcc222
  persistentVolumeReclaimPolicy: Retain
  storageClassName: efs-sc
  volumeMode: Filesystem
status:
  phase: Bound
---
apiVersion: v1
kind: PersistentVolume
metadata:
  name: airuntime-efs-pv
spec:
  accessModes:
  - ReadWriteMany
  capacity:
    storage: 30Gi
  claimRef:
    apiVersion: v1
    kind: PersistentVolumeClaim
    name: airuntime-efs-pvc
    namespace: runtime-build-space
  csi:
    driver: efs.csi.aws.com
    volumeHandle: fsap-07f23a2ea21fcc222
  persistentVolumeReclaimPolicy: Retain
  storageClassName: efs-sc
  volumeMode: Filesystem
status:
  phase: Bound

설치 명령어 실행​

아래 명령어로 설치를 수행한다.

./runtime-installer quick_install

설치는 다음 순서로 이루어진다.

  1. kfserving(kserve) 설치
    1. cert-manager 설치
    2. istio-metrics 설치
    3. knative 설치
    4. kfserving 설치
    5. fluent-bit 설치
    6. tekton-pipelines 설치
  2. runtime 설치
    1. 설정한 환경에 맞춰 기본 리소스 생성
    2. App 배포

이 과정에서 kserve라는 폴더가 생성되어, kfserving(kserve) 설치에 필요한 리소스를 담게 된다.

./hack
├── apps
├── database
├── install.env                          <--- Runtime 설치 환경 변수
├── kserve                               <--- KServe 설치용 폴더(자동 생성)
│   ├── cert-manager
│   ├── istio-metrics
│   ├── kfserving-installer
│   ├── kfserving.yaml
│   ├── knative-eventing
│   ├── knative-serving
│   ├── kserve_accuinsight_install.log    <--- KServe 설치 진행 로그
│   ├── kserve_accuinsight_setup.log      <--- KServe 설치 준비 로그
│   └── tekton
├── kfserving-installer
├── runtime-installer
└── runtime_install.log                   <--- Runtime 설치 진행 로그
